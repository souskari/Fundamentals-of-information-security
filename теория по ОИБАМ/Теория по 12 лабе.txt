Структура ACL (ACCESS CONTROL LIST) используется в операционной системе Windows для определения контроля доступа к объектам.
Она может быть представлена в виде списка элементов - ACE (ACCESS CONTROL ENTRY), каждый из которых определяет доступ к объекту для определенного субъекта с определенными правами доступа. 

СТРУКТУРА: 

1. Субъект (Subject): определяет, для какого пользователя или группы пользователей будет установлено правило доступа. 

2. Тип ACE (ACE Type): указывает тип элемента ACE. Существуют 4 типа ACE: Access Allowed ACE, Access Denied ACE, System Audit ACE, System Alarm ACE. Первые два типа определяют разрешения или запреты на доступ к объекту, оставшиеся два используются для проверки безопасности объектов в аудитории. ( ** Но по-моему на 2 последних в этой лабе вообще пофиг**) 

3. Права доступа (Access Rights): определяют, какие операции с объектом разрешены или запрещены. Например, чтение, запись, выполнение, удаление и т.д.

NTFS (New Technology File System) – это система файлов операционной системы Windows, которая используется для хранения и организации файлов на жестких дисках. (*** ОТВЕТ файловая система - тоже устроит***)
В NTFS используется древовидная структура каталогов для организации файлов. Каждый файл или папка имеют уникальный идентификатор. 

При входе в систему каждому пользователю назначается маркер доступа (access token). При запросе доступа компьютер сравнивает идентификаторы пользователя и групп пользователей из маркера доступа (токен, про который речь выше) с идентификаторами, хранящимися в ACE из ACL объекта.
